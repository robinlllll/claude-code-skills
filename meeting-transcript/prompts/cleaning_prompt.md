# Formatting Rules

1. **Header Injection**: 在文件最开头，必须插入一个【会议元数据】块，格式如下：
    ```
    【会议元数据】
    会议日期：{YYYY-MM-DD}
    文件名：{Original_Filename}
    核心议题：{详细列出3-5个核心讨论话题}
    涉及公司：{List_Corrected_Company_Names, 用**加粗**标注，附英文名}
    ---
    ```

2. **Company Name Formatting**: 首次出现的公司名请使用 **加粗** 并附英文：例如 **帝亚吉欧**（Diageo）。后续出现可只用中文名。

3. **Punctuation**: 原文可能缺乏标点，请根据语气逻辑添加逗号、句号和段落换行，使长篇大论变得可读。

4. **Speaker Labeling**: 使用 **加粗** 格式标注说话人：`**霖(00:00:03):**`。如果同一人连续发言被断行了，请合并。

5. **Minimal Filler Cleanup（保守清理）**:
   - **保留**所有说话人的对话，包括简短回应（"嗯嗯"、"对"、"好"）。这些回应体现了讨论的互动性，不要删除。
   - 仅修正同一句话内的**重复词**："就是就是"→"就是"，"我我"→"我"
   - 仅删除**ASR产生的句末乱码**（明显不是人说出来的随机字符）
   - **不要省略、缩写或总结任何段落**。保持原文的完整对话结构。
   - 目标：清洗后的文本应保持原文长度的80%以上。完整性优先于简洁性。

6. **[???] Markers**: 如果有一段话完全无法理解，请保留原文并在后面加 `[???]` 标记。**宁可标注不确定也不要强行猜测编造内容**。这非常重要——我需要知道哪些地方识别失败了。

7. **Sichuan Dialect**: 保留自然的四川口语表达（啥子、晓得、咋个、搞不赢、遭），这些是特色不是错误。

# Output
请直接输出修正后的完整文本内容，不要包含你的思考过程。
**关键要求：**
1. 确保处理完整个文件，不要中途截断。
2. 输出长度应≥原文长度的80%。不要大幅删减或概括内容。
3. 保留所有说话人的发言，包括简短的互动回应。
4. 完整性是最高优先级。
