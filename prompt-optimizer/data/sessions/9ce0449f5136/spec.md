# PromptSpec

## Goal
优化投资研究周会转写的总结prompt，生成结构化的公司分析纪要

## 任务一句话
将腾讯AI转写的投资周会实录（含识别错误），整理成按公司/行业分组的结构化分析纪要，保留主讲人的分析逻辑和操作倾向。

## 输入
- 输入格式：腾讯AI自动转录的会议文字，格式为 `说话人(时间戳): 内容`
- 输入特点：
  - 多人对话（主讲人 lilin + 其他参会者）
  - 包含识别错误（公司名、财务数据、语句断裂）
  - 涉及多个公司/行业（如 LVMH、星巴克、Deckers、烟草、地产建材等）
  - 时长约1-2小时，转写内容约5万字

## 输出
- 输出格式：
  1. **按公司/行业分组的纪要**：每家公司3-6段自然语言复述
  2. **四个锚点结构**：
     - 核心观点摘要（立场+逻辑）
     - 正文复述（自然语言重建分析链）
     - 潜在行动提示（明确方向+动作+理由）
     - 关键跟踪点（单列，每条≤10字）
  3. **一句话汇报摘要表**：表格形式，供投委会快速阅读

## 硬约束（必须满足）
- 按公司/行业分组，不能混在一起
- 每家公司必须包含四个锚点结构
- 潜在行动提示必须明确方向（偏多/中性/偏空）和动作（加仓/减仓/维持/观察）
- 关键跟踪点必须具体，每条≤10字
- 必须生成一句话汇报摘要表

## 软约束（尽量满足）
- 语气自然流畅，像真实研究助理整理的会议复盘
- 用"他说/他认为/他提到"的语气展开，忠实还原主讲人的思考
- 对同一公司重复提及的部分合并成完整逻辑
- 若出现行业链（如 homebuilders→materials→labor→rates），可跨公司串联

## 必须避免
- 不要机械分点，要自然语言复述
- 不要使用"若……则……"假设性描述（在潜在行动提示中）
- 不要主观加工，保留原始模糊与不确定性
- 不要遗漏主讲人提到的重要数据点
- 语义不明时不要强行解释，标注"语义不明"

## 评估标准
- 公司/行业分组是否完整准确
- 四个锚点结构是否齐全
- 潜在行动提示是否明确具体（方向+动作+理由）
- 关键跟踪点是否可跟踪、可验证
- 语言是否自然流畅，非机械条列
- 公司名/财务数据是否正确修复

## 测试用例
用 lilin 的投资周会转写（约5万字，涉及10+公司）测试：
- 检查是否正确识别所有公司/行业
- 检查四个锚点结构是否完整
- 检查潜在行动提示是否明确
- 检查一句话摘要表是否生成
